# 源码研读-2026-04-06

## SkillHub CLI 研读（~1551行 Python 单文件）

### 核心架构
**单文件 CLI**，纯标准库（无外部依赖），通过 `argparse subparsers` 实现4个子命令：search / install / upgrade / list / self-upgrade。采用模块级 `from skills_upgrade import cmd_upgrade` 解耦升级逻辑。

### 关键文件
- `skills_store_cli.py`：主程序，153行到~900行是 install/upgrade/search/lockfile 核心逻辑；900行~1100行是 OpenClaw plugin 检测和自动注入；1100行~1275行是 self-upgrade 逻辑（最复杂）
- `skills_upgrade.py`：独立升级模块，183行，职责单一
- `~/.skillhub/config.json` + `metadata.json`：运行时配置（URL模板、版本）

### 可复用点
1. **多层 fallback URL 设计**：install 支持双 URL 模板（主站 + COS 备份），download 时依次尝试，failover 自动切换
2. **self-upgrade 模式**：CLI 自己下载新版本到临时目录，校验 SHA256，然后原子性替换自身（通过 `sys.argv[0]` + subprocess re-exec）
3. **插件自动检测**：读取 `~/.openclaw/openclaw.json`，自动注入 skillhub plugin 配置，无需手动操作
4. **安全提取**：tempdir + safe_extract_zip 防止路径遍历

### 差距分析
SkillHub 极简（无框架、纯裸写），而我倾向引入库和框架；它的 self-upgrade + plugin auto-discovery 模式非常干净，我没有类似机制。
